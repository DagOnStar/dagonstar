# Teaching notes

Teach the numbered lessons in order: graph construction, execution and movement, structured task models, distributed infrastructure, then FAIR composition and CWL. Each lesson has one primary concept, an observable verification, a controlled experiment, and a scientific-practice discussion. Ask for a prediction before execution and distinguish observation (for example, an exit status or checksum) from inference (for example, portability or reproducibility).

Infrastructure lessons use structural checks before optional live checks. Do not replace those boundaries with simulated claims. When implementation and prose disagree, source and passing tests are authoritative; correct the lesson and record the limitation.

Use the same synthetic meteorological case study across the core sequence so
that changes in workflow semantics are not confounded with changes in subject
matter. Grade explanations against the dependency model and evidence, not
against incidental output formatting. Live infrastructure demonstrations are
enrichment activities and must not determine whether a learner can complete the
core course.

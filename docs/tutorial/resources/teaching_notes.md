# Teaching notes

Teach the numbered lessons in order: graph construction, execution and movement, structured task models, distributed infrastructure, then FAIR composition and CWL. Each lesson has one primary concept, an observable verification, a controlled experiment, and a scientific-practice discussion.

Infrastructure lessons use structural checks before optional live checks. Do not replace those boundaries with simulated claims. When implementation and prose disagree, source and passing tests are authoritative; correct the lesson and record the limitation.
